# Nginx Log Monitoring Lab

## Overview

This lab demonstrates the deployment and management of an NGINX web server within a Linux environment, focusing on web hosting and log monitoring activities.

The objective was to gain hands-on experience with web server configuration, website deployment, and real-time log analysis.

---

## Lab Environment

### Server

- Ubuntu Linux
- NGINX Web Server

### Client

- Remote User Access

---

## Tools Used

- NGINX
- Linux Terminal
- Tail Command
- System Logs

---

## Objectives

- Install and configure NGINX
- Deploy a simple web page
- Monitor web server activity
- Analyze server log entries

---

## Methodology

### Phase 1: NGINX Installation

NGINX was installed and configured on a Linux virtual machine.

Skills Demonstrated:

- Linux Administration
- Service Management
- Web Server Deployment

---

### Phase 2: Website Deployment

A sample HTML page was created and hosted on the NGINX server.

Skills Demonstrated:

- Web Server Configuration
- Content Hosting
- Directory Management

---

### Phase 3: Remote Access Testing

The hosted web page was accessed from another device to verify availability and connectivity.

Skills Demonstrated:

- Network Connectivity Testing
- Web Service Verification

---

### Phase 4: Log Monitoring

Real-time monitoring of NGINX logs was performed to observe user activity and web requests.

Skills Demonstrated:

- Log Analysis
- Event Monitoring
- Troubleshooting

---

## Key Learning Outcomes

- Linux System Administration
- NGINX Configuration
- Web Server Deployment
- Log Monitoring
- Event Analysis

---

## Security Relevance

Log monitoring plays an important role in security operations by allowing administrators to:

- Detect unauthorized access attempts
- Monitor user activity
- Investigate incidents
- Identify abnormal behavior
